## Contribution Strategy

### Phase 1 — Research (Day 1)
1. Browse all classmate repositories shared in WhatsApp/Zoom from Assignment 14
2. Read each project's CONTRIBUTING.md and README to understand setup requirements
3. Look for issues labelled `good-first-issue` or `help-wanted`
4. Comment on chosen issues saying "I would like to work on this!" before starting — this prevents two people doing the same work

### Phase 2 — Start Small (Day 2-3)
- Begin with documentation fixes or test additions — these are the easiest to get merged
- They also help me learn the codebase before tackling feature work
- Example: fix a typo in README, add a missing docstring, add a test for an untested function

### Phase 3 — Feature Contributions (Day 4-5)
- Once familiar with the codebase, tackle a `good-first-issue` feature
- Keep PRs small and focused — one PR = one change
- Write tests for any new code added
- Make sure CI passes before requesting review

### Phase 4 — Respond to Feedback (Day 6-7)
- Check PR comments daily
- Respond to review feedback within 24 hours
- Make requested changes on the same branch and push — the PR updates automatically

---

## Rules I Will Follow for Every PR

1. **Always comment on the issue first** before starting work
2. **Always create a new branch** — never work on main
3. **Always run the project's tests** before submitting
4. **Always write a clear PR description** linking to the issue
5. **Always keep PRs small** — one feature or fix per PR
6. **Never copy-paste code** without understanding it
